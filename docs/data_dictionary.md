# Data Dictionary — Last Mile Launch Readiness Program

## sites.csv
| Field | Description |
|---|---|
| site_id | Unique station identifier (DXP1–DXP5) |
| site_name | Station name |
| region | Geographic region |
| launch_date | Committed Day 1 launch date |
| site_type | Same-Day or Standard delivery station |

## stakeholders.csv
| Field | Description |
|---|---|
| stakeholder_id | Unique ID |
| name | Role title |
| department | Owning department |
| role | Functional role |
| raci_default | Typical RACI designation for this role |

## milestones.csv
| Field | Description |
|---|---|
| milestone_id | Unique ID |
| site_id | Associated site |
| phase | Launch playbook phase |
| planned_date | Planned completion date |
| owner_dept | Owning department |
| status | Complete / On Track / At Risk / Delayed / Blocked / Not Started |
| sequence | Order within the 9-phase playbook |

## tasks.csv
| Field | Description |
|---|---|
| task_id | Unique ID |
| site_id | Associated site |
| phase | Launch playbook phase |
| task_name | Task description |
| owner_dept | Owning department |
| due_date | Task due date |
| status | Complete / On Track / At Risk / Delayed / Blocked / Not Started |
| priority | High / Medium / Low |

## raid_log.csv
| Field | Description |
|---|---|
| raid_id | Unique ID |
| site_id | Associated site |
| type | Risk / Assumption / Issue / Dependency |
| description | Item description |
| category | Schedule / Operational / People / Technology / Facilities / Vendor / Safety |
| severity | High / Medium / Low |
| probability | High / Medium / Low |
| owner | Responsible role |
| mitigation | Mitigation approach |
| due_date | Resolution target date |
| status | Open / Mitigating / Closed / Escalated |

## raci_matrix.csv
Activity-by-role matrix. Each cell is R (Responsible), A (Accountable), C (Consulted), I (Informed), or blank (not involved).

## training.csv / hiring.csv
Per-site readiness against training and hiring targets, including completion percentages.

## equipment.csv
Per-site equipment install tracking (scanners, dock doors, conveyor, WiFi APs, badge readers) — planned vs. installed quantity and status.

## day1_validation.csv
Per-site Day 1 Readiness Checklist results — checklist item, Pass/Fail/Pending, owner, checked date, comments.

## budget.csv
Per-site planned vs. actual launch cost and variance.

## schedule_variance.csv
Per-site, per-phase planned vs. actual duration and variance in days.

## issues.csv
Post-launch and in-flight issues by site, department, and resolution status.

## site_scorecard.csv (generated by `dashboard/analytics.py`)
| Field | Description |
|---|---|
| readiness_score | 0–100, weighted blend of task completion (40%), milestone completion (35%), Day 1 validation pass rate (25%), minus open high-severity risk penalty |
| risk_score | 0–100, severity-weighted score of open risks |
| schedule_variance_days | Sum of schedule variance across all 9 phases |
| launch_confidence_score | Readiness score adjusted down for risk and schedule variance — the number that drives Go/No-Go |
| go_no_go | GO (≥80), CONDITIONAL GO (60–79), NO-GO (<60) |
| open_blockers | Count of tasks with status = Blocked |
| critical_risks | Count of open high-severity risks |
| delayed_milestones | Count of milestones with status = Delayed |
